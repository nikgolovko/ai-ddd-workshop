[.lead]
This document provides two things for Step 4 of the VQS DDD Workshop:

. A *blank canvas template* for a single bounded context (Section 1). Copy once per context.
. *Five fully worked canvases* for the VQS system (Section 2).

Duration: *90 minutes* total · ~18 minutes per canvas. +
Goal: Define the explicit boundaries, responsibilities, language, and team ownership of each bounded context. Start with the Core Domain.

=== Strategic Classification Reference

[cols="2,8", options="header"]
|===
| Classification | Meaning

| Core Domain
| Your primary competitive differentiator. Build in-house, invest deeply, protect from external coupling.

| Supporting Subdomain
| Necessary for the core to function but not differentiating. Build in-house but invest proportionally.

| Generic Subdomain
| Commodity capability. Consider buying, reusing, or using open-source before building.
|===

=== Integration Pattern Reference

[cols="2,8", options="header"]
|===
| Pattern | Meaning

| Customer-Supplier (C-S)
| Upstream produces, downstream consumes. Upstream may dictate the contract.

| Anti-Corruption Layer (ACL)
| Downstream translates upstream output into its own domain model. Upstream is untrusted.

| Open Host Service (OHS)
| Upstream publishes a stable, versioned API for multiple consumers.

| Published Language (PL)
| Agreed event schema shared across contexts. Loose coupling.

| Conformist
| Downstream adopts upstream's model without translation. High coupling — use sparingly.

| Partnership
| Two teams coordinate in lockstep. High coupling — use only when context boundaries are provisional.
|===

=== Canvas 1: Inspection (Edge)

==== Context Identity

[cols="2,5", options="header"]
|===
| Field | Value

| Context Name
| Inspection

| Strategic Classification
| *Core Domain* — primary competitive differentiator; real-time defect detection is the product

| Loop
| Decision Loop (edge)

| Team Ownership
| Edge Platform Team

| Team Topology Type
| Platform team — owns EV-07 edge node fleet, HMI firmware, and Inspection context software
|===

==== Purpose / Mission

____
Real-time detection of defects on production units at the edge: assigns a Decision to every InspectionCase using versioned DecisionBand thresholds, captures all Overrides with structured ReasonCodes, and forwards closed cases and DriftSignals to downstream contexts.
____

==== Key Capabilities

[cols="1,8", options="header"]
|===
| # | Capability

| 01 | Sub-200 ms inference on live conveyor frames via edge node EV-07
| 02 | DecisionBand lookup and auto-assignment per active DBV version
| 03 | HMI alert and operator review flow (thumbnail + ReasonCode gate)
| 04 | ReasonCode-gated Override capture — enforces AC-01 at HMI layer
| 05 | InspectionCase lifecycle management: open → reviewed → closed → forwarded
| 06 | DriftSignal publication on statistical override-rate anomaly
|===

==== Ubiquitous Language

[cols="3,6,3", options="header"]
|===
| Term | Definition within Inspection | Notes

| InspectionCase
| A single panel inspection event. Aggregates all decisions and overrides for one unit from conveyor arrival to case closure.
|

| Defect
| A detected anomaly on a production unit. Classified by DefectType and ConfidenceScore. May or may not result in SCRAP.
|

| Decision
| The outcome assigned to an InspectionCase: SCRAP, REWORK, or PASS. Derived from DecisionBand lookup.
|

| DecisionBand
| The active mapping from ConfidenceScore ranges to Decision categories. References a versioned DBV from Model Operations.
|

| ConfidenceScore
| The raw [0,1] model output. NOT a domain decision. Never displayed directly to the operator.
|

| Override
| A structured operator correction of a model Decision. Gated on ReasonCode selection. Creates an AuditEntry.
| ✅ new from Step 3

| ReasonCode
| A structured code from the active RCV version. Required to submit any Override. Enforced at HMI layer.
| ✅ new from Step 3

| DriftSignal
| A published event raised when override rate exceeds a statistical anomaly threshold. Consumed by Model Operations.
| ✅ new from Step 3

| HMIAlert
| The physical and digital notification fired when a Defect is detected. Presents thumbnail and override controls to operator.
|

| TemporaryBandOverride
| An emergency, time-limited adjustment to DecisionBand thresholds. Requires ML Engineer + Plant Manager approval. Not HMI-editable.
| ✅ new from Step 3
|===

==== Outbound Dependencies

[cols="3,3,3,5", options="header"]

| The unit of work triggered by an RCARequested event. Has its own lifecycle.

| A reference to an image batch, override log, or sensor reading used as evidence in hypothesis generation.

==== Key Capabilities

[cols="1,8", options="header"]

| 05 | Human approval gate for all Recommendations — mandatory before action

|===

==== Ubiquitous Language

[cols="3,6,3", options="header"]
|===
| Term | Definition within Case Management | Notes

| Case
| Lifecycle wrapper around one or more InspectionCases sharing a defect pattern. NOT the same as InspectionCase.
|

| Recommendation
| An ACL-translated, actionable instruction derived from RootCauseHypothesis. Requires human approval before any action.
| ✅ new from Step 3 (moved from RCA)

| ConfidenceBucket
| ACL mapping of LLMConfidence: High ≥0.80, Medium 0.50–0.79, Low <0.50. The only RCA-derived term allowed in this domain model.
| ✅ new from Step 3

| AuditEntry
| Immutable record of every state change. Contains actor, timestamp, action, linked ID, and RCV version if Override. EU AI Act Article 10 record.
|

| OverrideWindow
| Rolling 24 h window per DefectType counting model-error ReasonCategory Overrides toward the RCA trigger threshold.
| ✅ new from Step 3

| ReasonCodeVocabulary (RCV)
| Versioned set of valid ReasonCodes. Owned by this context. Current version: RCV-1 (6 codes). Reviewed quarterly.
| ✅ new from Step 3

| ReasonCategory
| model-error (RC-03, RC-04, RC-05) triggers RCA; material-variation (RC-01, RC-02) does not.
| ✅ new from Step 3

| EscalationFlag
| Set on Cases where RC-07 ('other') was used. Triggers mandatory supervisor review.
| ✅ new from Step 3

| CauseVocabularyGap
| Received from RCA via ACL. Holds Recommendation in 'unclassified' state pending vocabulary extension.
| ✅ new from Step 3
|===
